# orca.py sync Command Implementation Plan

> **For agentic workers:** Use `superpowers:test-driven-development` and `superpowers:verification-before-completion` while executing this plan.

**Goal:** Add an `orca.py sync` subcommand that updates `kvParam.nozzle_temperature` in custom 99xxx DB entries from matching OrcaSlicer user presets, per `docs/2026-07-13-sync-command-spec.md`.

**Architecture:** The new `sync` subcommand lives in `skill/orca.py` (it already owns preset reading and matching). It calls `cfs.py` via `subprocess` for `pull`, `edit`, `push`, and `verify` to avoid duplicating SSH/DB logic. Pure comparison and matching logic is extracted into small, testable helpers.

**Tech Stack:** Python 3, `argparse`, `json`, `subprocess`, `pytest`, existing `skill/orca.py` and `skill/cfs.py`.
