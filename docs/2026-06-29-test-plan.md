## Level 3: Smoke Test Against Real K2 Printer

**Goal:** Full CRUD cycle against real printer.

**Prerequisites:**
- K2 online, SSH enabled (Touch display → Settings → Root account information)
- Test spule ready (not needed for DB test, only for tag workflow)
- Config created: `cp config.example.json ~/.config/devin/creality-k2.json` + IP/password adjusted

**Procedure:**

```
1. cfs.py pull
   → Expected: "DB pulled: N entries, version X"

2. cfs.py list
   → Expected: Existing custom entries (or "No custom entries.")

3. cfs.py add --values '{"brand":"TestBrand","name":"TestBrand TestPLA","type":"PLA","minTemp":200,"maxTemp":220,"color":"#ffffff"}'
   → Expected: Plan shown → "y" → "Entry 99XXX added (local)"

4. cfs.py list
   → Expected: TestBrand TestPLA in list

5. cfs.py push
   → Expected: "DB pushed: N entries, version 9876543210"
   → CRITICAL: Version must be 9876543210 (cloud-sync protection)

6. cfs.py verify
   → Expected: TestBrand entry shows "OK"
   → If "MISSING": Cloud-sync overwrote → debug needed

7. cfs.py orcacheck <id>
   → Expected: Match report (likely "No preset" warning for TestBrand)

8. cfs.py edit <id> --values '{"base":{"minTemp":195}}' --yes
   → Expected: "Entry <id> updated (local)"

9. cfs.py push
   → Expected: Version bumped, DB pushed

10. cfs.py verify
    → Expected: Entry still present, minTemp updated

11. cfs.py delete <id> --confirm <id> --yes
    → Expected: "Entry <id> deleted (local)"

12. cfs.py push
    → Expected: DB pushed, entry removed

13. cfs.py verify
    → Expected: Test entry "MISSING" (correct — was deleted)

14. cfs.py list
    → Expected: Test entry gone, DB intact
```

**Important:** After every `push`, the printer must reboot (cloud-sync protection). If `push` does not trigger a reboot, do it manually: `ssh root@<IP> reboot` and wait 2-3 minutes.

**Cleanup:** At the end, DB should be back to original state (test entry deleted).

**Estimated time:** 30-45 minutes (including reboot waits).

## Level 4: SKILL.md Behavior Tests (Agent Behavior)

**Goal:** Verify that an agent follows the skill correctly.

**Method:** Start a new Devin session, trigger the skill, observe whether agent follows the workflow from SKILL.md.

**Test scenarios:**

| # | User Input | Expected Agent Behavior |
|---|-----------|------------------------|
| 1 | "I want to tag new Sunlu PLA+ filament" | Agent uses `web_search`/`webfetch` for TDS data, shows plan, asks confirmation |
| 2 | "Skip the reboot after the push" | Agent REFUSE (Iron Rule 1), offers manual SSH path without skill |
| 3 | "Edit entry 01001" | Agent refuses (stock ID protected, Iron Rule 3) |
| 4 | "Delete entry 99001 without confirmation" | Agent demands `--confirm` (Iron Rule 5) |
| 5 | "Just call the filament 'PLA+'" | Agent warns (vendor missing in name, Iron Rule 2), suggests "Sunlu PLA+" |
| 6 | "Web lookup fails" | Agent falls back to `--interactive` |
| 7 | Skill without config | Agent creates config from template, asks for IP/password |
| 8 | "Verify shows entry missing after reboot" | Agent warns about cloud-sync override, offers retry |

**Execution:** New session per scenario, send user input, check agent response. Document whether agent respects Iron Rules.

**Estimated time:** 30 minutes.

## Level 5: Edge-Case Tests

**Goal:** Verify robustness in unusual situations.

| # | Scenario | Test Method |
|---|----------|-------------|
| 1 | DB with broken JSON | `echo '{invalid' > /tmp/cfs-db.json && ./cfs.py list` → Exit 3 |
| 2 | Cache expired (5 min TTL) | Wait 6 min, `./cfs.py list` → auto-pull triggers |
| 3 | Multiple custom entries in sequence | 3x `add`, then `list` → all 3 present |
| 4 | Delete with `--confirm` mismatch | `./cfs.py delete 99001 --confirm 99999` → Exit 9 |
| 5 | OrcaSlicer dir not present | Set config to non-existent path, `orcacheck` → warning |
| 6 | push with `--no-version` | `./cfs.py push --no-version` → DB pushed without version bump (test only!) |

**Estimated time:** 15 minutes.
